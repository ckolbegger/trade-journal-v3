# Implementation Plan — Index

Vertical-slice delivery plan. Each slice ships functionality the trader can see and use. Each slice is a series of user stories; each story is a set of checkboxed tasks with TestSpecs, ending in live browser verification. The design being implemented lives in [docs/design/](../design/) and [docs/adr/](../adr/).

## The story loop

Every story is a self-contained loop an implementer (human or agent) runs to completion:

1. Implement the story's tasks in order.
2. Unit + integration tests green (`npm test`).
3. Run the app (`npm run dev`).
4. Drive the story's **Browser Verification** steps in a real browser; fix and repeat until every observation matches.
5. Encode the happy path as one Playwright test; full Playwright suite green (`npm run test:e2e`).
6. Check every box. A story with an unchecked box is not done.

Edge cases live in Vitest (unit/integration). Playwright carries exactly one happy-path test per story — the regression loop for later stories.

## Conventions (fixed for all slices)

- **Stack**: Vite + React + TypeScript (strict), Dexie over IndexedDB, Vitest (+ fake-indexeddb for integration), Playwright for e2e. React Router for pages.
- **Money** is integer cents (`type Money = number`, always whole cents). Never float dollars. Display formatting is a UI concern.
- **Qty** is a positive integer (shares or contracts); direction lives on the Execution's `side`.
- **Dates**: `ISODate` is `'YYYY-MM-DD'` (trading date); `Timestamp` is epoch ms. No timezone math in v1 — the trader's local date is the trading date.
- **InstrumentKey** is a canonical string: `"AAPL"` for stock; `"AAPL 2028-01-15 C 220"` for contracts (OCC-style, strike in dollars).
- **IDs** are `crypto.randomUUID()` strings, generated by the Books.
- **The fact contract lives in `src/domain/`.** TradeMath's parameter types (`TradeRecord`, `LegFacts`, `ExecutionFacts`, `ExitLevel`, `Instrument`, `Mark`, `MarkSet`, `MarkSeries`, …) are defined in the domain layer; Books import their record shapes *from* domain, never the reverse — that direction is what makes "TradeMath imports nothing" enforceable. Types TradeMath never computes over stay in their module: `Entry`/`EntryType`/`Anchor` in Journal, `FetchReport` in PriceBook, `PlanDraft`/`ExecutionOutcome` in TradeBook.
- **Module layout** (mirrors [design/overview.md](../design/overview.md)):
  ```
  src/
    domain/trademath/     pure functions + domain types (no imports from elsewhere in src/)
    storage/              StorageBinding interface, in-memory + Dexie implementations, Dexie schema
    books/tradebook/      TradeBook over a StorageBinding
    books/journal/        Journal over a StorageBinding
    books/pricebook/      PriceBook over a StorageBinding (+ PricingSource adapters, Slice 4)
    coordinators/         Valuations, Review
    workspace/            Workspace (seeding, settings, export/import)
    ui/                   React app: pages, components, composition root
  tests/integration/      cross-module tests over Dexie + fake-indexeddb
  e2e/                    Playwright specs, one per story
  ```
- **Dependency rules** are law: TradeMath imports nothing; Books call only their StorageBinding (+ TradeMath for TradeBook detection, Slice 9); coordinators call Books + TradeMath; UI calls Books for writes/fact-reads and coordinators for anything computed — never TradeMath or StorageBindings directly. Enforce by review at every story; an ESLint boundary rule is added in S0.1.
- **Interfaces are implemented subset-first**: a slice implements only the operations its stories need, but signatures come verbatim from the design docs so later slices add operations without reshaping.
- **Just-in-time construction.** Nothing is built ahead of the story that needs it — no speculative types, type-union variants, result fields, tests, or "shape declared for later" stubs. When a later slice needs something, that slice's story adds it. If deferring something would force a later rewrite, the slice file says so explicitly (e.g., Slice 1's scaling note) — pull-forwards are a flagged exception, never a default.
- **Test layers**: unit = module against in-memory binding (fast, exhaustive, edge cases); integration = same behavior over Dexie + fake-indexeddb (persistence round-trips, indexes, transactions); e2e = one Playwright happy path per story.
- **E2e is opt-in, never part of the default run.** `npm test` runs unit + integration only (what `npm run build` / pre-commit gates on). Browser verification is invoked explicitly afterward: `npm run test:e2e` (Playwright starts its own dev server via `webServer` config), or as a separate CI step. The story loop still *requires* running it before checking a story's boxes — it's manual-invoke, not optional.
- **TestSpec format**: each task lists `describe` / `it` blocks. The `it` lines are the contract — an implementer writes exactly these tests (plus any they find necessary), then makes them pass.
- **Checkbox protocol**: `- [ ]` → `- [x]` only when the code exists, its TestSpec tests pass, and (for verification tasks) the browser run succeeded. The slice's row in the table above flips to ☑ when every story in it is checked.

## Decided-in-plan semantics

Small rulings the design docs left open, fixed here so stories are unambiguous (worked examples in the slice files):

- **Fees**: each Execution carries total fees (cents). `Valuation.fees` = sum of all fees on the Trade. Realized P&L is net of the fees on the Executions that opened and closed the realized quantity; unrealized P&L is gross (no projected exit fees). `totalPnL = realizedPnL + unrealizedPnL`.
- **`RiskReward.original`**: measured from actual entry basis to the *original* Plan's stop/target; `'undefined'` until the first Execution exists.
- **Abandoned planned Trade**: `statusOf` returns `'closed'` when a Trade has no Executions but a Close Reason (ADR 0005's explicit-close of a dead thesis).
- **Seed content v1** (trader-editable from Slice 13 on): Strategy "Long Stock"; Close Reasons Hit Target / Hit Stop / Thesis Invalidated / Timed Out / Never Filled; Entry Types "Plan", "Close", "Trade Review" (Action list: Hold / Exit Soon / Adjust / Watch Closely), "Review Note", "Trader Reflection". Each item is seeded in the slice where the trader can first legitimately select it and a test can exercise that selection ("Rolled" arrives with Slice 16's roll gesture; Review Note / Trader Reflection with Slice 2) — `ensureSeeded` is additive-iff-absent, so late-arriving defaults reach old databases for free.
