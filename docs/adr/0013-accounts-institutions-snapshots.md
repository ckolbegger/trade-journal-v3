# Accounts bind Positions; value arrives as dated snapshots

The trader holds multiple Accounts across multiple Institutions. Every Position belongs to exactly one Account, and Transfers (campaign restructuring) only connect Positions in the same Account — moving holdings between brokerages is a different real-world event and will be modeled separately if ever needed. Account value is never derived: an Account Snapshot records (account, date, total liquidation value, optional note), entered opportunistically. Cumulative P&L curves per account derive from Executions and Marks regardless; the snapshot series adds the true equity curve and %-of-account risk framing at whatever cadence the trader sustains. A full cash ledger (deposits/withdrawals) was rejected — it turns the journal into bookkeeping software and diverges the moment anything happens outside the app; snapshots therefore cannot separate performance from contributions, which is accepted.
